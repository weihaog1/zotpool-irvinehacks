### Task 1: Install Tailwind CSS properly and remove CDN

Currently Tailwind is loaded via CDN `<script>` in index.html. Install it as a proper dependency.

**Files:**
- Modify: `package.json`
- Modify: `vite.config.ts`
- Modify: `index.html`

**Steps:**

1. Install dependencies:
```bash
npm install tailwindcss @tailwindcss/vite
```

2. Update `vite.config.ts` to add the Tailwind plugin:
```typescript
import tailwindcss from '@tailwindcss/vite';
```

```css
@import "tailwindcss";
```

7. Run `npm run dev` and verify the app still renders correctly with all styles intact.

## Phase 4: Matching

### Task 16: Implement matching algorithm

**Files:**
- Create: `services/matching.ts`

**Steps:**

1. Install date-fns:
```bash
npm install date-fns
```

2. Create `services/matching.ts` implementing the scoring formula from PRD section 13:
   - `calculateMatchScore(rideA, rideB): MatchScore`
   - Route proximity (40%): Haversine distance between origins + destinations
   - Time overlap (25%): departure window compatibility
   - Day overlap (20%): matching scheduled days
   - Preferences (15%): gender + cost compatibility
   - Hard filters: same type, gender violation, zero day overlap, >5mi origins

3. `findMatches(userRide, allRides): Array<{ride: Ride, score: MatchScore}>` - returns ranked list.

4. Quality labels: >= 80 "Excellent match", >= 60 "Good fit", < 60 "Possible match".

5. Commit: `feat: Implement client-side ride matching algorithm`
